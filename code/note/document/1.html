# vue-router

# 动态路由匹配

# 动态路由使用

  • "路径参数"使用冒号:标记

    { "path": "/user/:id", "component": User }

  • 当匹配到一个路由时，参数值会被设置到 this.$route.params

    <div>User{{ $route.params.id }}</div>
"

  • 可以设置多段"路径参数"

    /user/:username/post/:post_id

{ username: 'evan', post_id: 123 }

# 响应路由参数变化

  • watch 监测$route 对象

    const User = {
  template: "...",
  watch: {
    $route(to, from) {
      // 对路由变化作出响应...
    },
  },
};

  • 引入 beforeRouteUpdate 守卫(2.2 版本)

    const User = {
  template: "...",
  beforeRouteUpdate(to, from, next) {
    next();
  },
};

# 高级匹配模式

  • 使用 path-to-regexp 作为路径匹配引擎，可以进行正则匹配

  • 可选的动态路径参数、匹配零个或多个、一个或多个，甚至是自定义正则匹配

# 嵌套路由

  • 可以通过路由的 children 属性配置下级路由

  • 下级路由

    • 对应在当前渲染组件的<router-view>中

    • 以/开头的嵌套路径为根路径，只需写下级路由短路径

    • 当下级路由 path:""时，可通过根路径触发

# 编程式导航

  • 在导航方法中，onComplete 和 onAbort 属于 2.2 版本

  • <router-link>会在内部调用 router.push(...)

  • 主要 api

    router.push(location，onComplete，onAbort)

// location可为字符串或对象
// 如果提供了path参数，则params参数无效（query例外）
// 如果是命名路由，则params参数有效
// path参数支持path:`/user/${变量}`的写法

router.replace(location，onComplete，onAbort) // 同上

router.go(n) // n为整数

# 重定向和别名

# 重定向

  • 通过 routes 配置中的 redirect 参数完成

  • redirect 属性支持字符串(路径和 name)和方法

  • redirect 为方法时，参数为目标路由，return 为重定向的字符串路径/路径对象

# 别名

  • 别名用 alias 参数表示，意味着访问别名路径，同样能访问该组件

  • 访问别名的变化，可以用 watch->$route 和 beforeRouteUpdate 方法监测到

  • 别名用字符串(路径)和数组表示

# 路由组件传参

const User = {
    props: ['id'],
    template: '<div>User {{ id }}</div>'
}
routes: [
    { path: '/user/:id', component: User, props: true }, 		// 布尔模式
    { path: '/user/:id', component: User, props: { id: 1 } },	// 对象模式(path变化已无效，传递值已固定)
    { path: '/user/:id', component: User, props: (route) => {	// 函数模式1
        return { name: route.params.id + '_name'}
    }}
    { path: '/user', component: User, props: (route) => {		// 函数模式2
        return { query: route.query.xx };
    }},
]

# HTML5 History 模式

  • 通过 mode 属性进行设置

    const router = new VueRouter({
    mode: 'history',

  • 最好在服务端增加一个覆盖所有情况的候选资源，前端页做一个覆盖所有情况的 404 组件页

# 导航守卫

  • 守卫种类

    • 全局守卫——路由实例

    • 路由守卫——路由对象(配置)

    • 组件守卫——组件页面

  • 守卫钩子

// 路由实例
beforeEach((to, from, next) =>{})
beforeResolve((to, from) =>{})		// (2.5版本+)
afterEach((to, from) =>{})

// 路由对象
beforeEnter(to, from, next) =>{}

// 组件页面
beforeRouteEnter(to, from, next) =>{}
beforeRouteUpdate(to, from, next) =>{}	 // (2.2版本+)
beforeRouteLeave(to, from, next) =>{}

  • 守卫顺序

    1. 离开页面前——beforeRouteLeave
    2. 全局前置守卫——beforeEach
    3. 重用组件触发——beforeRouteUpdate
    4. 路由配置触发——beforeEnter
    5. 解析异步路由组件
    6. 进入页面前——beforeRouteEnter
    7. 导航确认前——beforeResolve
    8. 全局后置守卫——afterEach
    9. 触发 DOM 更新
    10. 新的组件实例传递给 beforeRouteEnter 中的 next 参数

  • 注意点

    1. 导航在所有守卫 resolve 完才会执行，所以相应钩子的 next 参数函数必须执行，否则导航会一直等待
    2. 参数或查询的改变并不会触发进入/离开的导航守卫，可以用 watch->$route 和 beforeRouteUpdate 方法监测到
    3. beforeRouteEnter 钩子意味着，组件实例还没被创建，不能访问 this，支持给 next 传递回调的唯一守卫，参数为 vm
    4. beforeResolve 钩子意味着，导航确认前，所有组件守卫和异步路由组件被解析之后

# 数据获取

# 导航完成前获取

beforeRouteEnter (to, from, next) {
    getPost(to.params.id, (err, post) => {
        next(vm => vm.setData(err, post))   // 组件实例还未创建，无this
    })
}

// 复用组件时的导航变化
beforeRouteUpdate (to, from, next) {
    this.post = null
    getPost(to.params.id, (err, post) => {
        this.setData(err, post)		        // 组件实例已经创建，有this
        next()
    })
}

# 导航完成后获取

afterEach((to, from) => {
    this.ajaxData();
})

// 此时data已经被observed(观察)
created () {
   this.fetchData()
}

# 滚动行为

  • scrollBehavior 方法只在支持 history.pushState 接口的游览器上有效

  • savedPosition 参数代表最后操作的位置，只在 popstate 触发操作上可用

# 示例代码

const router = new VueRouter({
    routes: [...],
    scrollBehavior (to, from, savedPosition) {
        // return值为期望滚动到的位置，为空(falsy值)则不滚动
        // return格式为{ selector: string, offset? : { x: number, y: number }}
        // offset只在2.6+支持
    }
})

# 模拟滚动到锚点

scrollBehavior (to, from, savedPosition) {
    if (to.hash) {
        return {
            selector: to.hash
        }
    }
}

# 异步滚动(2.8+)

scrollBehavior (to, from, savedPosition) {
    return new Promise((resolve, reject) => {
        setTimeout(() => {
            resolve({ x: 0, y: 0 })
        }, 500)
    })
}

# 路由懒加载

  • 基于 vue 的异步组件和 webpack 代码分割功能

    • vue 组件支持以一个工厂函数方式定义组件，这个函数会异步解析组件定义(接受 Promise 返回)

    • webpack 支持 code-splitting 功能，2.0 版本更支持动态 import

      Vue.component('xx', function (resolve) {
    // 方式1
    resolve({ template: '<div>I am async!</div>' })

    // 方式2——webpack会自动切包，然后通过ajax请求加载
    require(['./my-async-component'], resolve)

    // 方式3——返回一个能resolve组件的Promise
    return import('./my-async-component')
}

new Vue({
    // 同上
    components: {
        "my-component": () => import('./my-async-component')
    }
})

  • 把组件按组分开

    • 把某个路由下的所有组件都打包在同个异步块(chunk)中

    • 原理是利用 webapck 的注释语法来提供 chunk name

      const Foo = () => import(/* webpackChunkName: "group-foo" */ "./Foo.vue");
const Bar = () => import(/* webpackChunkName: "group-foo" */ "./Bar.vue");

  • 如果想在 network 中看到整合名称，需要在 webpack.base.conf.js 的 output 中增加 chunkFilename

    output: {
  chunkFilename: "[name].js";
}
