# 前端框架的知识构成

# js 运行时特性

  • 异步特性——让 js 代码有序运行，合理更新，避免阻塞和卡顿
  • 闭包——合理缓存，缓存运行时和组件数据
  • 原型——合理发挥 js 的继承特性
  • 上下文——合理递归，不爆栈
  • 作用域链——避免多余跨越，减少引擎步长

# js 宿主拓展

  • 发挥 dom
  • 发挥 bom
  • 发挥 web api

# 编程思想

  • 通过设计模式，解决代码规模问题
  • 通过编程范式，解决建模问题

# 计算机基础

  • 模拟计算机编译原理，进行框架运行时的编译，实现代码转换
  • 通过合理的数据结构，保证数据的内存合理分配，有效分配
  • 通过算法，避免冗长执行路径，提升 CPU 运算效率
